# 新功能规划

## 优先级说明

- **P1**: 高优先级 - 对核心功能有重要影响
- **P2**: 中优先级 - 提升用户体验或性能
- **P3**: 低优先级 - 锦上添花

---

## P1 - 高优先级

### 1. Avellaneda-Stoikov 做市执行器

**状态**: 部分完成

**描述**: 实现基于 Avellaneda-Stoikov 模型的做市执行器，动态计算最优 bid/ask spread。

**已完成**:
- ✓ TradeIntensityIndicator - 订单强度估计（已重构到 hft/indicator/intensity.py）
- ✓ IntensityResult - 强度计算结果（buy_k, sell_k, imbalance）
- ✓ get_optimal_spread() - 最优价差计算

**待完成**:
- AS 执行器整合（使用 TradeIntensityIndicator）
- 实时库存调整
- 动态 gamma 参数

**相关文件**:
- `hft/executor/avellaneda_stoikov.py` (待整合)
- `hft/indicator/intensity.py` (已完成 - TradeIntensityIndicator)

### 2. 风控模块

**状态**: 待实现

**描述**: 添加独立的风控 Listener，监控并限制风险暴露。

**功能**:
- 仓位限制（单品种/总仓位）
- 日内亏损限制
- 波动率过滤（暂停高波动期交易）
- 流动性检查

### 3. 回测框架

**状态**: 待实现

**描述**: 支持策略回测，使用历史数据模拟交易。

**功能**:
- 统一数据接口（实盘/回测）
- 模拟撮合引擎
- 性能指标计算（夏普、最大回撤等）

## P2 - 中优先级

### 4. 策略热重载

**状态**: 待实现

**描述**: 支持运行时更新策略配置，无需重启。

**实现思路**:
- 文件监听器检测配置变化
- 策略 `on_reload(new_config)` 回调
- 平滑过渡（避免订单冲突）

### 5. Web 监控界面

**状态**: 待实现

**描述**: 添加 Web 界面实时展示系统状态。

**功能**:
- 仓位/余额展示
- 订单历史
- 策略性能图表
- 告警配置

### 6. 多策略资金分配

**状态**: 待实现

**描述**: 自动在多策略间分配资金。

**功能**:
- 固定比例分配
- 凯利公式动态分配
- 策略绩效加权
